# ex05_if_else

## Goal

Practice conditional logic with `if`, `else if`, and `else` by reading an integer and reporting whether it is negative, zero, or positive.

## Initial Problems

Initial buggy version:

```c
#include <stdio.h>

int main(void) {
        int number;
        printf("Input a number:");
        scanf(%d, &number);
        if(&number < 0 {
                printf("Your number is negative\n");
        } else if(&number  == 0) {
                printf("Your number is zero\n");
        }else {
                printf("Your number is positive\n");
        };
        return 0;
}
```

Problems:
- `scanf(%d, &number);` used `%d` without quotes
- `if(&number < 0)` compared the address instead of the value
- the first `if` condition was missing a closing `)`
- there was a stray `;` after the conditional block

## Safer Final Version

```c
#include <stdio.h>

int main(void) {
        int number;

        printf("Input a number: ");

        if (scanf("%d", &number) != 1) {
                printf("Invalid input\n");
                return 1;
        }

        if (number < 0) {
                printf("Your number is negative\n");
        } else if (number == 0) {
                printf("Your number is zero\n");
        } else {
                printf("Your number is positive\n");
        }

        return 0;
}
```

## Example Runs

Valid input:

```text
Input a number: 5
Your number is positive
```

Invalid input:

```text
Input a number: hello
Invalid input
```

## Lessons Learned

- `scanf()` format strings must be quoted
- comparisons must use the variable value, not its address
- syntax errors in conditions are often small but fatal
- input should be validated before the program uses it
- `scanf()` return values matter for safe behavior
