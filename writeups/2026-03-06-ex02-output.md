# ex02_output

## What I Built

A small C program that:
- declares a character array initialized with a string literal
- declares an integer variable
- prints both values using `printf()`
- produces two lines of output

## Problems I Encountered

### 1. Tried to use `str`

Initial idea:

```c
str name = "...";
```

Problem:
- C does not have a built-in `str` type

Fix:

```c
char name[] = "...";
```

### 2. Incorrect format specifiers

Problem:
- mixed up `%d` and `%s`

Why it matters:
- `printf()` is variadic, so format mismatches are not just incorrect output
- they can cause undefined behavior

Correct usage:
- `%s` for C strings
- `%d` for integers

## Lessons Learned

- C strings are typically stored as `char[]`
- `printf()` requires correct format specifiers
- the compiler can catch many type mismatches when warnings are enabled
- simple programs are useful for reinforcing syntax and structure

## Final Program

```c
#include <stdio.h>

int main(void) {
        int age = 27;
        char name[] = "Matahari Ramadhan";
        printf("My Name: %s\n", name);
        printf("My Age: %d\n", age);
        return 0;
}
```
