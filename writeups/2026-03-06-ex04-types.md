# ex04_types

## Goal

Write a small program that declares several basic C types and prints them with the correct `printf()` format specifiers.

## Initial Problems

### 1. Character literal error

Initial code:

```c
char aChar = a;
```

Problem:
- the compiler treated `a` as an identifier, not a character literal

Fix:

```c
char aChar = 'a';
```

Characters use single quotes.

### 2. Incorrect format specifier for a C string

Initial code:

```c
printf("%c is string\n", aString);
```

Problem:
- `%c` is for a single character
- `aString` is a character array used as a C string

Fix:

```c
printf("%s is string\n", aString);
```

## Final Code

```c
#include <stdio.h>

int main(void) {
        int number = 23;
        char aChar = 'a';
        char aString[] = "This is String";
        float phi = 3.14f;
        printf("%d is integer\n", number);
        printf("%c is character\n", aChar);
        printf("%s is string\n", aString);
        printf("%f is float\n", phi);
        return 0;
}
```

## Example Output

```text
23 is integer
a is character
This is String is string
3.140000 is float
```

## Lessons Learned

- each C type must match the correct `printf()` format specifier
- characters and strings use different literal syntax
- C strings are typically represented as `char[]`
- compiler warnings are useful for catching type mismatches early
