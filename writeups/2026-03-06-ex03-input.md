# ex03_input

## Goal

Practice reading user input with `scanf()` and printing the result with `printf()`.

## Initial Bug

Initial code:

```c
#include <stdio.h>

int main(void) {
        int number;
        printf("Input number:");
        scanf("Input number: %d", &number);
        printf("Your number: %d", number);
        return 0;
}
```

Observed result:

```text
Input number:8
Your number: 0
```

## Root Cause

The `scanf()` format string was incorrect:

```c
scanf("Input number: %d", &number);
```

This tells `scanf()` to match the literal text `Input number: ` before reading the integer. A normal user only types the numeric input, so the pattern does not match and the read fails.

## Fix

Separate the prompt from the input format:

```c
printf("Input number: ");
scanf("%d", &number);
```

`printf()` displays the prompt. `scanf()` reads only the expected value.

## Important Follow-Up

The basic fix above is still incomplete for safe C code.

Reason:
- `scanf()` returns the number of successfully matched inputs
- for one integer, success means the return value must be `1`
- if input fails and `number` is printed anyway, `number` may be uninitialized

Safer pattern to adopt:
- check the return value of `scanf()`
- handle failure before using `number`

## Lessons Learned

- `scanf()` reads according to the format string exactly
- prompts belong in `printf()`, not in `scanf()`
- `scanf()` needs the address of the destination variable
- input functions must be checked before trusting the result

## Rule to Remember

```text
printf() prints formatted output
scanf() reads input into addresses
```
